# Build AI Meeting Notes with LiveKit and AssemblyAI Universal-3 Pro

Build a Zoom-inspired video meeting app with an AI notetaker that joins as a visible participant, transcribes speech in real-time using AssemblyAI Universal-3 Pro, responds via voice, and generates structured meeting notes when the session ends.

### ❶ Install dependencies

<codefocussection filepath="server/requirements.txt" filerange="1-3" title="Python dependencies" themeColor="#0000FF" label="Server">

Install the LiveKit Agents SDK with plugins for each service: AssemblyAI for speech-to-text, OpenAI-compatible client for the Cerebras LLM, Rime for text-to-speech, and Silero for voice activity detection. Also install `livekit-plugins-noise-cancellation` for built-in noise cancellation and `python-dotenv` for environment variable loading.

```bash
pip install "livekit-agents[assemblyai,openai,silero,rime]" livekit-plugins-noise-cancellation python-dotenv
```

### ❷ Configure API keys

<codefocussection filepath="server/.env.example" filerange="1-5" title="Environment variables template" themeColor="#0000FF" label="Server">

Create a `.env` file with your API keys. You'll need keys for [AssemblyAI](https://www.assemblyai.com/) (speech-to-text), [Cerebras](https://cloud.cerebras.ai/) (LLM), and your [LiveKit Cloud](https://cloud.livekit.io/) project credentials (URL, API key, and secret). The client also needs a `.env.local` with the same LiveKit credentials plus the Cerebras key for post-meeting notes generation.

</codefocussection>

### ❹ Define meeting keyterms

<codefocussection filepath="server/meeting_notes.py" filerange="19-33" title="Define meeting keyterms" themeColor="#0000FF" label="Server">

Define keyterms that AssemblyAI's Universal-3 Pro will use to boost recognition accuracy for meeting-specific vocabulary. These cover common meeting terms (action items, deliverables, quarterly review, roadmap) plus product names. U3P's `keyterms_prompt` appends these to its built-in default prompt, improving transcription of domain-specific vocabulary without replacing the optimized turn-detection behavior.

</codefocussection>

### ❺ Define the meeting facilitator persona

<codefocussection filepath="server/meeting_notes.py" filerange="35-55" title="Define the meeting facilitator persona" themeColor="#0000FF" label="Server">

The `SYSTEM_INSTRUCTIONS` string defines the AI's personality for the LLM — a meeting facilitator that summarizes discussion points, asks clarifying questions, flags action items, and keeps responses concise for voice delivery.

The `MeetingFacilitator` class extends LiveKit's `Agent` base class and passes these instructions to the parent constructor.

</codefocussection>

### ❻ Configure STT, LLM, TTS, and start the session

<codefocussection filepath="server/meeting_notes.py" filerange="58-102" title="Configure and start the agent session" themeColor="#0000FF" label="Server">

The `entrypoint` function is the core of the agent. After connecting to the LiveKit room, it sets up each component:

**STT (AssemblyAI U3P):** Configured with tuned turn detection — `end_of_turn_confidence_threshold=0.4` sets how confident U3P must be that the speaker has finished before triggering a turn end, `min_turn_silence=400` waits 400ms of silence before checking for an end of turn, and `max_turn_silence=1280` forces a turn end after 1.28 seconds. `vad_threshold=0.3` with a matching `activation_threshold=0.3` on the VAD ensure the agent responds quickly. These thresholds balance responsiveness with accuracy while U3P's punctuation-based detection prevents mid-sentence cutoffs.

**LLM (Cerebras):** Uses `llama3.1-8b` via the OpenAI-compatible API wrapper with a temperature of 0.7 for natural-sounding responses.

**TTS (Rime):** The `mistv2` model with the `astra` speaker voice. `reduce_latency=True` prioritizes fast first-byte delivery.

**Turn detection:** Set to `"stt"` so AssemblyAI's punctuation-based turn detection drives the conversation flow instead of relying solely on VAD silence thresholds.

**Noise cancellation:** The session starts with `RoomOptions` that enable LiveKit's BVC (Background Voice Cancellation) on the audio input, filtering out background noise before it reaches the STT.

After starting the session, `generate_reply()` makes the AI introduce itself as a meeting facilitator and ask about the agenda — so the AI speaks first when a participant joins.

</codefocussection>

### ❼ Generate LiveKit room tokens

<codefocussection filepath="client/src/app/api/token/route.ts" filerange="1-45" title="Token API route" themeColor="#0000FF" label="Client">

This API route generates a LiveKit access token when the user clicks "Start Meeting." It creates a unique room name and participant identity, then uses `RoomServiceClient` to create the room and dispatch the agent before returning the token — so the agent starts booting while the client connects. The token grants `canPublish: true` and `canSubscribe: true` so the participant can share both audio and video.

</codefocussection>

### ❽ Generate meeting notes from transcript

<codefocussection filepath="client/src/app/api/notes/route.ts" filerange="1-85" title="Notes generation API route" themeColor="#0000FF" label="Client">

When the user clicks "End Meeting," the client sends the accumulated transcript to this API route. It calls the Cerebras LLM with a structured prompt that asks for a JSON response containing a summary, key points, decisions, and action items. The route includes fallback JSON parsing in case the LLM wraps the response in extra text. Since the transcript includes speaker labels (`User:` and `AI Notetaker:` prefixes), the LLM can distinguish who said what.

</codefocussection>

### ❾ Build the Zoom-inspired meeting UI

<codefocussection filepath="client/src/app/page.tsx" filerange="118-554" title="Video meeting layout with transcript" themeColor="#0000FF" label="Client">

The client connects to LiveKit with both `audio={true}` and `video={true}`, enabling the user's webcam alongside their microphone. An `AgentGate` component waits for the AI agent to join the room before showing the meeting UI, displaying a loading state in the meantime. `RoomAudioRenderer` ensures the user hears the AI's voice. The `MeetingView` component creates the Zoom-inspired layout:

**Video grid:** Two tiles side-by-side — the user's webcam (via `VideoTrack`) and the AI notetaker represented by an animated cloud orb whose glow and scale react to the agent's audio level.

**Live caption bar:** Shows the most recent transcription line in a Zoom-style closed-captions overlay with color-coded speaker labels.

**Transcript panel:** A scrolling log of all messages from both the user and AI, sorted chronologically. Partial (interim) transcripts appear dimmed in gray as the user speaks, then update in place when finalized. The `buildConversation` helper merges `agentTranscriptions` (from `useVoiceAssistant()`) and `userSegments` (from a custom `useUserTranscription` hook that listens to `RoomEvent.TranscriptionReceived`) into a single timeline.

**Control bar:** Mic toggle, camera toggle, and "End Meeting" button using `localParticipant.setMicrophoneEnabled()` / `setCameraEnabled()` from the LiveKit SDK.

</codefocussection>

### ❿ Run the app

Start the server and client in separate terminals:

```bash
# Terminal 1 — Python agent
cd server
python meeting_notes.py dev
```

```bash
# Terminal 2 — Next.js client
cd client
npm install && npm run dev
```

Open [http://localhost:3000](http://localhost:3000), click **Start Meeting**, and grant camera/microphone permissions. The AI facilitator will introduce itself and ask about the agenda. Speak naturally — the transcript updates in real-time with speaker labels. Click **End Meeting** to generate structured notes with a summary, key points, decisions, and action items.
